# TESTING.md

## How to run

```bash
python -m pytest            # full suite
python -m pytest tests/test_preprocessing.py -v
python scripts/verify_preprocessing.py   # phase-1 verification loop evidence (real page)
```

## What is tested

| Layer | Where | Depends on |
|---|---|---|
| Unit tests per stage | `tests/test_<stage>.py` | synthetic fixtures only (regenerable, committed) |
| Real-page verification per phase | `scripts/verify_<stage>.py` | `tests/fixtures/real_sample_page.jpg` + ground truth |
| End-to-end integration test | arrives with phase 5 (JSON export) | real page |

Synthetic fixtures are generated by `tests/fixtures/make_synthetic.py` (deterministic,
seeded) and committed, so tests don't depend on regeneration:

- `synthetic_simple.png` — clean two-column page: headline band, 2 body columns, caption line.
- `synthetic_noisy.png` — same layout + Gaussian noise, salt & pepper, illumination gradient,
  ~1.5° rotation, black scanner-edge bars.

## Evaluation metrics (phase 6 target definitions)

- **Region precision/recall**: greedy one-to-one matching of detected boxes to GT boxes by
  IoU, match threshold IoU ≥ 0.5. Precision = matched/detected, recall = matched/GT.
- **Word accuracy**: per matched region, `1 - word_level_levenshtein(gt, ocr)/len(gt_words)`,
  aggregated over matched regions weighted by GT word count. Threshold in `config.yaml`
  (`verification.ocr.word_accuracy_threshold`, currently 0.70).
- **Fragmentation rate**: average number of detected boxes overlapping (IoU ≥ 0.1) each GT
  region — 1.0 is ideal; >1 means articles are being split.
